# 创建自定义增援

如果您有兴趣在战斗中召唤精英来增援首领，本指南将详细介绍这一点。

## 创建自定义首领的步骤：

1. 创建您的主要自定义首领，它将召唤您的自定义增援。
2. 为增援创建自定义首领。请记住，自定义首领可以比原版生物更强，也可以更弱。
3. 在配置文件中添加正确的增援行。

## 创建增援行

假设您有一个自定义首领，其格式如下：

```yaml
isEnabled: true
entityType: ENDER_DRAGON
name: $eventBossLevel &eBinder of Worlds
level: 200
powers:
- ender_dragon_arrow_bombardment.yml
```

并且您想为其添加增援。以下是一个增援的示例：

```yaml
- summonType:ON_COMBAT_ENTER
  filename: binder_of_worlds_phase_1_parkour_reinforcement.yml
```

要使其生效，请将其添加到 powers 列表中：

```yaml
isEnabled: true
entityType: ENDER_DRAGON
name: $eventBossLevel &eBinder of Worlds
level: 200
powers:
- ender_dragon_arrow_bombardment.yml
- summonType: ON_COMBAT_ENTER
  filename: binder_of_worlds_phase_1_parkour_reinforcement.yml
```

### chance

这是一个可选的配置字段，用于补充 `summonType` 的 `ONCE` 和 `ON_HIT` 选项。它表示首领受到攻击时自定义增援生成的几率。

| Key      |        Values         | Default |
|----------|:---------------------:|:-------:|
| `chance` | [Double](#double) |  none   |

<details>

<summary><b>示例</b></summary>

<div align="left">

```yml
chance: 0.5
```
*表示有 50% 的几率生成自定义增援。*

</div>

</details>

***

### location

`location: x,y,z` 是一个可选的配置字段，允许管理员将增援的位置偏移一定数量的方块。以下是您需要了解的关于此选项的关键概念：

**偏移位置是相对的。** 这意味着 `location: 0,10,0` 将在首领正上方 10 个方块处生成首领，而不是在世界坐标 0,10,0 处。

**区域首领的相对位置不同。** 区域首领将始终使用其生成点作为相对位置的原点，这意味着如果您知道首领的生成点在哪里并计算方块数，您可以让增援首领在特定的方块位置生成。Binder of Worlds 的巢穴就依赖于此，无论巨龙首领在哪里，都能在极其特定的方块位置生成挑战塔首领。

**如果您测量距离出错，首领会生成在方块内部。**

| Key        |      Values       | Default |
|------------|:-----------------:|:-------:|
| `location` | [String](#string) |  none   |

<details>

<summary><b>示例</b></summary>

<div align="left">

```yml
location: 0,10,0
```
*对于非区域自定义首领，无论它在哪里，增援都将在首领正上方 10 个方块处生成。*

*对于区域自定义首领，无论首领在哪里，增援都将在区域首领的生成点正上方 10 个方块处生成。*

</div>

</details>

_**重要提示：对于 `summonType: ON_COMBAT_ENTER_PLACE_CRYSTAL` 来说，这是一个强制字段！**_

### lightningRod

它不能用于其他自定义增援，因为那些应该依靠 powers 来实现这类能力。

| `lightningRod` | [Boolean](#boolean) | `false` |

### spawnNearby

| Key           |       Values        | Default |

| `spawnNearby` | [Boolean](#boolean) | `false` |

## 组合一个大型增援

让我们使用之前的大部分字段创建一个示例增援：

```yaml
- summonType: ON_COMBAT_ENTER
  filename: test_boss.yml
  spawnNearby: true
  inheritAggro: true
  inheritLevel: true
```

这会执行以下操作：

- 创建一个自定义增援。
- 当自定义首领进入战斗时，该自定义增援将生成 (`summonType: ON_COMBAT_ENTER`)。
- 召唤的自定义增援将是 `test_boss.yml` 中的那个 (`filename: test_boss.yml`)。
- 自定义增援将在自定义首领当前位置的 30 个方块范围内生成 (`spawnNearby: true`)。
- 自定义增援将继承其增援的自定义首领的仇恨，这意味着它将攻击造成最多伤害的玩家 (`inheritAggro: true`)。
- 无论 `test_boss.yml` 中的设置如何，自定义增援将与增援它的自定义首领具有相同的等级 (`inheritLevel: true`)。

```yaml
- summonType: ON_COMBAT_ENTER_PLACE_CRYSTAL
  location: -80,10,0
  lightningRod: true
```

这会执行以下操作：

- 创建一个自定义增援。
- 该自定义增援将是一个末影水晶，在战斗开始时被放置 (`summonType: ON_COMBAT_ENTER_PLACE_CRYSTAL`)。
- 末影水晶将延迟地在其周围地面上降下闪电 (`lightningRod: true`)。

现在将其应用于自定义首领：

```yaml
isEnabled: true
entityType: ENDER_DRAGON
name: $eventBossLevel &eBinder of Worlds
level: 200
powers:
- ender_dragon_arrow_bombardment.yml
- summonType: ON_COMBAT_ENTER
  filename: test_boss.yml
  spawnNearby: true
  inheritAggro: true
  inheritLevel: true
- summonType: ON_COMBAT_ENTER_PLACE_CRYSTAL
  location: -80,10,0
  lightningRod: true
```

您就完成了！这里还有一些额外的提示：

- 字段可以按任何顺序设置，但建议始终将 `summonable` 放在第一位！
- 这是对旧增援系统的重大改进，旧系统已不再列出。目前不建议依赖它，因为它将来可能会停止工作。
