Вот подробный разбор того, как масштабируется здоровье / урон в EliteMobs. Он должен использоваться в качестве справочника при проектировании подземелий.

Он ориентирован на пользовательских боссов, но обычные элиты масштабируются таким же образом.

# Масштабирование (в общем)

В EliteMobs по умолчанию боссы получают 7 единиц максимального здоровья и наносят на 1 единицу больше урона за уровень.

Оружие игроков наносит на 7 единиц больше урона за уровень, а броня снижает на 0.25 урона за уровень (но есть 4 слота брони, которые вносят вклад в общее значение, что составляет 1). Наконец, игроки получают максимальное здоровье по мере открытия престижных рангов.

# Урон, наносимый пользовательскими боссами

Формула урона для пользовательских боссов такова:

```
( ( ( базовый урон / 2 ) * множитель урона пользовательского босса) + уровень босса - снижение урона броней игрока ) * глобальный множитель урона
```

## базовый урон

Это может быть одно из двух:

1.  Значение в mobproperties для типа сущности босса.
2.  Поскольку об этом может быть трудно помнить при проектировании подземелий и другого контента, пользовательские боссы могут иметь поле normalizedCombat, которое применяет стандартный урон в 5 единиц, который имеет наилучшее масштабирование для прогрессии EliteMobs. В более позднем обновлении normalized combat будет включен по умолчанию для всех региональных боссов.

## уровень босса

## снижение урона броней игрока

Броня игрока снижает урон на 0,25 за уровень, и есть 4 слота брони, которые могут вносить вклад в общее значение. Кроме того, каждый уровень защиты от снарядов снижает урон от снарядов на 0,025.

Это множитель в MobCombatSettings. По умолчанию равен 1.

# Максимальное здоровье пользовательских боссов

Формула здоровья для пользовательских боссов такова:

```
( базовое максимальное здоровье + уровень босса * 7 ) * множитель здоровья пользовательского босса
```

## базовое максимальное здоровье

Это может быть одно из двух:

1.  Максимальное здоровье сущности Minecraft по умолчанию.
2.  Поскольку об этом может быть трудно помнить при проектировании подземелий и другого контента, пользовательские боссы могут иметь поле normalizedCombat, которое применяет стандартное здоровье в 7 единиц, которое имеет наилучшее масштабирование для прогрессии EliteMobs. # Урон, наносимый пользовательским боссам

Формула урона, с которым игроки бьют пользовательских боссов, имеет слишком много мелких переменных, таких как критические удары и перезарядки, чтобы подробно объяснять ее здесь. Упрощенная версия выглядит так:

```
урон = Уровень оружия
```

Примерно так должен выглядеть урон в идеальных средних условиях.

# Использование этих данных для подземелий

**ПРИМЕЧАНИЕ: ПРЕДПОЛАГАЕТСЯ, ЧТО ВЫ ИСПОЛЬЗУЕТЕ ЗНАЧЕНИЯ `normalizedCombat`!** Эти значения были специально созданы для более легкой балансировки подземелий.

Часть сложности босса заключается в том, чтобы сделать его здоровье и урон подходящими для схватки. Однако имейте в виду, что это лишь часть, и что способности так же важны.

## Легкие быстрые расчеты

1.  EliteMobs имеет 7 ударов в качестве количества ударов, которые требуются игроку с оружием того же уровня, что и босс, чтобы убить его, при условии отсутствия модификаторов.
    1.  Это означает, что множитель 2.0 означает, что игрокам придется ударить босса 2 x 7 = 14 раз. 0.5 означает 7 / 2 = 3.5.
2.  Базовый урон normalizedCombat, если броня игрока того же уровня, что и босс, составляет 2.5 урона или 1.25 сердец урона за удар.
    1.  Это означает, что множитель 2.0 означает, что игроки будут терять 3 сердца за удар, а множитель 0.5 заставит игроков терять 0.75 сердца за удар.

**Имейте в виду, что игроки, как правило, отдают приоритет оружию, и часто в итоге имеют гораздо худшую броню, чем оружие. Если вы не проектируете очень специфические типы арен с боем на расстоянии, вам, вероятно, не следует наносить большой урон за удар.** Протестируйте свой контент!

## Создание обычного моба для подземелий

При масштабировании normalizedCombat вы можете применить очень минималистичную конфигурацию:

```yaml
normalizedCombat: true
healthMultiplier: 1.0 #4 удара для убийства
damageMultiplier: 1.0 #1.5 сердца урона
```

**Примечание: поскольку 1.0 является значением по умолчанию, это означает, что вы также можете просто ничего не определять**.

## Создание пачек мусора

Пачки мусора многочисленны, но не очень опасны:

```yaml
normalizedCombat: true
healthMultiplier: 0.7 #3 удара для убийства
damageMultiplier: 0.5 #0.5 сердца урона
```

## Создание подкреплений

Подкрепления должны умирать очень легко, но представлять опасность с точки зрения урона (эти значения рекомендуются для 4+ сущностей ближнего боя, фактическое использование может отличаться):

```yaml
normalizedCombat: true
healthMultiplier: 0.25 #1 удар для убийства
damageMultiplier: 0.6 #1 сердце урона
```

## Создание мини-боссов

Мини-боссы должны держаться на своем месте и представлять механическую проблему для игроков. Это должно быть испытание навыков, которое длится недолго, но не должно быть чем-то чрезвычайно смертоносным:

```yaml
normalizedCombat: true
healthMultiplier: 3.0 #10 ударов для убийства
damageMultiplier: 1.2 #2 сердца урона
```

## Создание боссов

Боссы - это настоящий вызов, кульминация накопления в подземелье и настоящее испытание навыков, когда на карту поставлено все. Схватки должны быть долгими, и смерть должна быть постоянной угрозой.

```yaml
normalizedCombat: true
healthMultiplier: 7.0 #23 удара для убийства
damageMultiplier: 1.4 #2.5 сердца урона
```

## Создание мировых боссов

Мировые боссы опасны, но прежде всего у них много здоровья, и они призывают много подкреплений / имеют много атак по области. Они предназначены для сражения большими группами игроков в течение самого длительного времени, в течение которого должна длиться элитная схватка. Они не очень смертоносны, но имеют много механик, которые могут подавить неподготовленных игроков.

```yaml
normalizedCombat: true
healthMultiplier: 40.0 #160 ударов для убийства
damageMultiplier: 1.5 #2.5-3 сердца урона
```

Это всего лишь грубые рекомендуемые значения, и любые окончательные значения должны быть скорректированы с учетом самих схваток.
