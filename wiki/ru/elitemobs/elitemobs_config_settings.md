# Настройки EliteMobs по умолчанию

**EliteMobs поставляется с рекомендуемыми настройками по умолчанию. Если вы будете менять что-то случайным образом,
скорее всего, вы только ухудшите общее впечатление.**

_**EliteMobs предоставляется всем пользователям с рекомендуемыми настройками по умолчанию. Если вы будете менять что-то
случайным образом и затем запросите поддержку, велика вероятность того, что вам скажут сбросить файлы конфигурации.
Изменения плагина должны основываться на тестировании игрового процесса, а не на догадках.**_

## Начало работы

В настоящее время EliteMobs предлагает широкий спектр параметров конфигурации из-за высокого спроса на кастомизацию.
Чтобы эта вики-страница была краткой, будут отображаться только настройки по умолчанию.

Если вы хотите узнать больше о конкретной настройке, вы можете обратиться к файлу конфигурации и просмотреть комментарии.

## config.yml

`config.yml` содержит в основном случайные настройки, которые не могли удобно разместиться в других местах на момент их
создания. Это файл конфигурации, который, скорее всего, будет меняться между версиями.

<div align="left">

<details>

<summary><b>Развернуть таблицу</b></summary>

```yml
# Устанавливает языковой файл, используемый EliteMobs

# Определяет, будут ли элиты и боссы, созданные elitemobs, всегда показывать свои имена.
# Не рекомендуется!

superMobStackAmount: 50
# Заставляет элиты не наносить урон пассивным мобам взрывами.
preventEliteCreeperDamageToPassiveMobs: true
# Устанавливает, будет ли EliteMobs использовать заголовки для предупреждения игроков об отсутствующих разрешениях.
useTitlesForMissingPermissionMessages: true
# Устанавливает, будет ли EliteMobs предотвращать преобразование именованных мобов в элиты.
# Особенно важно для совместимости с другими плагинами.

# Устанавливает, будет ли EliteMobs преобразовывать мобов с пользовательскими причинами появления.
# Запуск определенных плагинов боссов автоматически установит это значение в true, независимо от значения в конфигурации.

nightmareWorldSpawnBonus: 0.5

# Настоятельно рекомендуется!

# Устанавливает, будет ли регенерация взрыва также восстанавливать содержимое контейнеров, таких как сундуки.

# Устанавливает ограничение символов на строку для страниц книжного меню.
# Уменьшите это значение, если текст обрезается в книжных меню, например, для квестов
characterLimitForBookMenuPagesV2: 170
# Устанавливает, будет ли пустое пространство меню заполнено стеклянными панелями.
# Не рекомендуется, если вы используете ресурспак EliteMobs.
useGlassToFillMenuEmptySpace: false
# Устанавливает, будет ли использоваться юникод для форматирования ресурспака EliteMobs.

# Устанавливает сообщение, отправляемое игрокам, которые открывают сундуки с сокровищами, уровень которых превышает ранг их гильдии.

# Устанавливает сообщение, которое появляется, рекомендуя команду /em alt для игроков, у которых есть проблемы с командой /em
dismissEMMessage: '&8[EliteMobs] &2/elitemobs &fменю не работает для вас? Попробуйте &2/elitemobs
  alt &fчтобы увидеть альтернативную версию меню! &cНе хотите видеть это сообщение
  снова? &4/em dismiss'
# Устанавливает сообщение, которое появляется, когда игроки выполняют команду /em alt.
switchEMStyleMessage: '&8[EliteMobs] &2/elitemobs &fстиль меню изменен! Посмотрите!'
```

## AntiExploit.yml

`AntiExploit.yml` позволяет администраторам настраивать различные аспекты системы AntiExploit. Система AntiExploit
существует для того, чтобы предотвратить использование игроками таких вещей, как фермы мобов, для легкого получения
монет и добычи, и подталкивает их к реальным сражениям и мини-подземельям вместо того, чтобы афк-кликами прокладывать
себе путь на вершину плагина.

Это критически важная система, которую нужно оставить включенной, если вы не хотите, чтобы люди просто афк фармили весь
плагин, и хотите, чтобы они действительно взаимодействовали с мини-подземельями.

## CombatTag.yml

`CombatTag.yml` содержит настройки конфигурации для системы Combat Tag. Эта система еще далека от завершения, но она
пытается управлять поведением игрока после того, как он вступил в бой.

<details>

<summary><b>Развернуть таблицу</b></summary>

```yml
# Устанавливает, включен ли боевой тег.
# Когда этот параметр включен, летающие игроки, вступающие в бой, перестают летать.
Enable combat tag: true
# Устанавливает сообщение, отправляемое при активации боевого тега.
Combat tag message: '&c[EliteMobs] Боевой тег активирован!'
# Устанавливает, будет ли у команды /ag таймер перед телепортацией.
Enable adventurers guild teleport timer: true
# Устанавливает сообщение действия, установленное во время ожидания таймера телепортации.
Teleport time left: '&7[EM] Телепортация через &a$time &7секунд...'
# Устанавливает сообщение, отправляемое, когда игроки перемещаются во время ожидания телепортации.
Teleport cancelled: '&7[EM] &cТелепортация прервана!'
```

</details>

## EconomySettings.yml

`EconomySettings.yml` содержит настройки конфигурации для экономики EliteMobs.

<details>

<summary><b>Развернуть таблицу</b></summary>

```yml
# Устанавливает, включена ли экономика EliteMobs. Это означает элитные монеты, возможность покупать и продавать снаряжение и возможность повышать ранги гильдий.
```
