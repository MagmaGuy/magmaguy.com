# Стандартные настройки EliteMobs

**EliteMobs поставляется со стандартными рекомендуемыми настройками. Если вы будете менять настройки наугад, то, скорее всего, только ухудшите общее впечатление от игры.**

_**EliteMobs предоставляется всем пользователям с рекомендуемыми настройками по умолчанию. Если вы меняете настройки наугад, а затем запрашиваете поддержку, существует большая вероятность того, что вам посоветуют сбросить файлы конфигурации. Модификации плагина должны быть основаны на тестировании в игре, а не на догадках.**_

## Начало работы

EliteMobs в настоящее время предлагает широкий спектр параметров конфигурации из-за высокого спроса на кастомизацию. Чтобы эта страница вики была краткой, будут показаны только настройки по умолчанию.

Если вы хотите узнать больше о конкретной настройке, вы можете обратиться к файлу конфигурации и просмотреть комментарии.

## config.yml

`config.yml` содержит в основном случайные настройки, которые не подходили для других файлов на момент их создания. Это файл конфигурации, который чаще всего претерпевает изменения между версиями.

<div align="left">

<details>

<summary><b>Развернуть таблицу</b></summary>

```yml
# Задает языковой файл, используемый EliteMobs

# Определяет, будут ли элиты и боссы, созданные elitemobs, всегда отображать свои имена.
# Не рекомендуется!
alwaysShowEliteMobNameTags: false
# Задает количество пассивных мобов, которые должны находиться в непосредственной близости, прежде чем они могут быть объединены.
# Не устанавливайте значение 0!
superMobStackAmount: 50
# Делает так, чтобы элиты не наносили урон пассивным мобам взрывами.
preventEliteCreeperDamageToPassiveMobs: true
# Устанавливает, будет ли EliteMobs использовать заголовки для предупреждения игроков об отсутствии разрешений.
useTitlesForMissingPermissionMessages: true
# Устанавливает, будет ли EliteMobs предотвращать превращение названных мобов в элитных.
# Особенно важно для совместимости с другими плагинами.
preventEliteMobConversionOfNamedMobs: true
# Устанавливает, будет ли EliteMobs преобразовывать мобов с нестандартными причинами появления.
# Запуск определенных плагинов боссов автоматически установит это значение в true, независимо от значения в конфигурации.
enableHighCompatibilityMode: false
# Устанавливает бонус к спавну элиты для кошмарного игрового режима
nightmareWorldSpawnBonus: 0.5
# Устанавливает, открывает ли централизованная команда /em главную страницу статуса плагина.
# Настоятельно рекомендуется!
emLeadsToStatusMenu: true
# Устанавливает, будут ли определенные команды, такие как /em wallet, вести к меню /em, где эта информация централизована.
otherCommandsLeadToEMStatusMenu: true
# Устанавливает, завершена ли настройка.

# Устанавливает местоположение спавна сервера по умолчанию для EliteMobs. /em spawntp приведет к этому месту.

# Устанавливает, будет ли EliteMobs восстанавливать блоки, взорванные элитами.
doExplosionRegen: true
# Устанавливает, будет ли регенерация взрыва также восстанавливать содержимое контейнеров, таких как сундуки.
# Установка значения false приведет к тому, что взрывы элиты не будут взрывать контейнеры.

defaultTransitiveBlockLimiter: 500
# Устанавливает, будет ли меню /em использовать только стиль меню на основе инвентаря, который совместим с Bedrock.
# Напоминаем, что игроки могут ввести команду /em alt для переключения между стилями меню /em
onlyUseBedrockMenus: false
# Устанавливает ограничение по количеству символов в строке для страниц меню книг.
# Уменьшите это значение, если текст обрезается в меню книг, например, для квестов
characterLimitForBookMenuPagesV2: 170
# Устанавливает, будет ли пустое пространство меню заполняться стеклянными панелями.
# Не рекомендуется, если вы используете пакет ресурсов EliteMobs.
useGlassToFillMenuEmptySpace: false
# Устанавливает, будет ли юникод использоваться для форматирования пакета ресурсов EliteMobs.
# Не устанавливайте это вручную, оно устанавливается автоматически при установке пакета ресурсов.
# Устанавливайте это вручную только в том случае, если вам пришлось объединить пакет ресурсов EliteMobs, и ожидайте, что интервалы могут не работать, если вы это сделаете.

# Устанавливает сообщение, отправляемое игрокам, если они вводят '/em confirm' без ожидающих команд.

# Устанавливает сообщение отслеживания для боссов, отправляющих сообщения отслеживания.
trackMessage: Отслеживать $name
# Устанавливает сообщение, отправляемое игрокам, открывающим сундуки с сокровищами с требованиями к уровню выше их ранга гильдии.
chestLowRankMessage: '&7[EM] &cВаш ранг гильдии должен быть не ниже $rank &c, чтобы
  открыть этот сундук!'
# Устанавливает сообщение, отправляемое игрокам, пытающимся повторно открыть сундук с сокровищами, который они уже открыли.
chestCooldownMessage: '&7[EM] &cВы уже открывали этот сундук недавно! Подождите $time!'
# Устанавливает сообщение, которое появляется с рекомендацией команды /em alt для игроков, у которых проблемы с командой /em.
dismissEMMessage: '&8[EliteMobs] &2/elitemobs &fменю не работает для вас? Попробуйте &2/elitemobs
  alt &f, чтобы увидеть альтернативную версию меню! &cНе хотите видеть это сообщение
  снова? &4/em dismiss'
# Устанавливает сообщение, которое появляется, когда игроки вводят команду /em alt.
switchEMStyleMessage: '&8[EliteMobs] &2/elitemobs &fстиль меню изменен! Посмотрите на него!'
```

```yml
disableCommonerRank: false
```

## AntiExploit.yml

`AntiExploit.yml` позволяет администраторам настраивать различные аспекты системы AntiExploit. Система AntiExploit существует для предотвращения использования игроками таких вещей, как фермы мобов, для легкого фарма монет и добычи, и подталкивает их к настоящим боям и мини-подземельям вместо того, чтобы просто кликать, чтобы попасть на вершину плагина.

Эту критически важную систему необходимо поддерживать включенной, если вы не хотите, чтобы люди просто фармили плагин в режиме AFK и хотите, чтобы они взаимодействовали с мини-подземельями.

## CombatTag.yml

`CombatTag.yml` содержит настройки конфигурации для системы Combat Tag. Эта система еще далека от завершения, но она пытается управлять поведением игрока после того, как он вступает в бой.

<details>

<summary><b>Развернуть таблицу</b></summary>

```yml
# Устанавливает, включен ли Combat Tag.
# При включении летающие игроки, вступившие в бой, перестают летать.
Enable combat tag: true
# Устанавливает сообщение, отправляемое при активации Combat Tag.
Combat tag message: '&c[EliteMobs] Combat tag активирован!'
# Устанавливает, будет ли у команды /ag таймер перед телепортацией.
Enable adventurers guild teleport timer: true
# Устанавливает сообщение о действии, установленное во время ожидания таймера телепортации.
Teleport time left: '&7[EM] Телепортация через &a$time &7секунд...'
# Устанавливает сообщение, отправляемое, когда игроки перемещаются во время ожидания телепортации.
Teleport cancelled: '&7[EM] &cТелепортация прервана!'
```

</details>

## EconomySettings.yml

`EconomySettings.yml` содержит настройки конфигурации для экономики EliteMobs.

<details>

<summary><b>Развернуть таблицу</b></summary>

```yml
# Устанавливает, включена ли экономика EliteMobs. Это означает элитные монеты, возможность покупать и продавать снаряжение, а также возможность повышать ранги гильдии.
# Если отключено, игроки не смогут прогрессировать в плагине!
enableEconomy: true
# Устанавливает стоимость перепродажи предметов в % от первоначальной цены. 5 означает 5%
itemResaleValue: 5.0
# Устанавливает внутриигровое название используемой валюты.
currencyName: Элитные монеты
```

## events.yml

`events.yml` содержит параметры конфигурации для событий.

_**Примечание: это только самые основные глобальные настройки.**_

Вы можете дополнительно изменить конкретные события в папке `events`.

<details>

<summary><b>Развернуть таблицу</b></summary>

```yml
# Определяет, будут ли события транслироваться только в мире, в котором происходят события.
Only broadcast event message in event worlds: false
# Устанавливает минимальный интервал в минутах между таймерными событиями.
actionEventMinimumCooldownMinutes: 240
# Определяет, будут ли происходить события действия.
# https://github.com/MagmaGuy/EliteMobs/wiki/Creating-Custom-Events#action-events
actionEventsEnabled: true
# Определяет, будут ли происходить таймерные события.
# https://github.com/MagmaGuy/EliteMobs/wiki/Creating-Custom-Events#timed-events
timedEventsEnabled: true
```

</details>
