# Значения конфигурации EliteMobs по умолчанию

**EliteMobs поставляется с рекомендованными настройками, установленными по умолчанию. Если вы измените что-то случайным образом, скорее всего, вы только ухудшите общий опыт.**

_**EliteMobs предоставляется всем пользователям с рекомендованными настройками по умолчанию. Если вы измените что-то случайным образом, а затем обратитесь за поддержкой, есть очень большая вероятность, что вам скажут сбросить файлы конфигурации. Изменения в плагине должны быть внесены на основе игрового тестирования, а не просто догадок.**_

## Начало работы

EliteMobs в настоящее время предлагает широкий спектр вариантов конфигурации из-за высокого спроса на настройку. Чтобы эта страница вики была краткой, будут показаны только конфигурации по умолчанию.

Если вы хотите узнать больше о конкретной настройке, вы можете обратиться к файлу конфигурации и просмотреть комментарии.

## config.yml

`config.yml` содержит в основном случайные настройки, которые не могли удобно разместиться в другом месте на момент их создания. Это файл конфигурации, который, скорее всего, будет изменен в разных версиях.

<div align="left">

<details>

<summary><b>Развернуть таблицу</b></summary>

```yml
# Устанавливает языковой файл, используемый EliteMobs.

# Устанавливает количество пассивных мобов, которые должны находиться в непосредственной близости, прежде чем их можно будет объединить.

superMobStackAmount: 50
# Делает так, чтобы элиты не наносили урон пассивным мобам взрывами.
preventEliteCreeperDamageToPassiveMobs: true
# Устанавливает, будет ли EliteMobs использовать заголовки для предупреждения игроков об отсутствующих разрешениях.
useTitlesForMissingPermissionMessages: true
# Устанавливает, будет ли EliteMobs предотвращать преобразование именованных мобов в элиты.
# Особенно важно для совместимости с другими плагинами.
preventEliteMobConversionOfNamedMobs: true
# Устанавливает, будет ли EliteMobs преобразовывать мобов с пользовательскими причинами появления.
# Запуск определенных плагинов боссов автоматически установит это значение в true, независимо от значения в конфигурации.

nightmareWorldSpawnBonus: 0.5
# Устанавливает, открывает ли централизованная команда /em главную страницу состояния плагина.

# Устанавливает, будет ли меню /em использовать только стиль меню на основе инвентаря, который совместим с Bedrock.

# Устанавливает ограничение количества символов в строке для страниц меню книги.

# Устанавливает, будет ли пустое пространство меню заполнено стеклянными панелями.

# Устанавливает, будет ли использоваться unicode для форматирования пакета ресурсов EliteMobs.

# Устанавливает сообщение, отправляемое игрокам, которые открывают сундуки с сокровищами с требованиями к уровню выше их ранга гильдии.

# Устанавливает сообщение, которое появляется, рекомендуя команду /em alt для игроков, у которых возникают проблемы с командой /em.
dismissEMMessage: '&8[EliteMobs] &2/elitemobs &fменю не работает? Попробуйте &2/elitemobs

## AntiExploit.yml

`AntiExploit.yml` позволяет администраторам настраивать различные аспекты системы AntiExploit. Система AntiExploit существует для того, чтобы игроки не могли использовать такие вещи, как гриндеры мобов, для легкого фарма монет и добычи, и подталкивает их к реальным боям и мини-подземельям вместо того, чтобы фармить в режиме AFK.

Это критически важная система, которую нужно держать включенной, если вы не хотите, чтобы люди просто фармили весь плагин в режиме AFK, а хотите, чтобы они взаимодействовали с мини-подземельями.

## CombatTag.yml

`CombatTag.yml` содержит настройки конфигурации для системы Combat Tag. Эта система все еще далека от завершения, но она пытается управлять поведением игрока после того, как он вступает в бой.

<details>

<summary><b>Развернуть таблицу</b></summary>

```yml
# Устанавливает, включена ли метка боя.
# Если включено, летающие игроки, вступающие в бой, перестают летать.
Enable combat tag: true
# Устанавливает сообщение, отправляемое при активации метки боя.
Combat tag message: '&c[EliteMobs] Метка боя активирована!'
# Устанавливает, будет ли у команды /ag таймер перед телепортацией.
Enable adventurers guild teleport timer: true
# Устанавливает сообщение действия, заданное во время ожидания таймера телепортации.
Teleport time left: '&7[EM] Телепортация через &a$time &7секунд...'
# Устанавливает сообщение, отправляемое, когда игроки двигаются во время ожидания телепортации.
Teleport cancelled: '&7[EM] &cТелепортация прервана!'
```

</details>

## EconomySettings.yml

`EconomySettings.yml` содержит настройки конфигурации для экономики EliteMobs.

<details>

<summary><b>Развернуть таблицу</b></summary>

```yml
# Устанавливает, включена ли экономика EliteMobs. Это означает элитные монеты, возможность покупать и продавать снаряжение и возможность повышать ранги гильдии.
# Если отключено, игроки не смогут прогрессировать в плагине!
enableEconomy: true
# Устанавливает цену перепродажи предметов в процентах от первоначальной цены. 5 - это 5%.
itemResaleValue: 5.0

## events.yml

Вы можете дополнительно изменить конкретные события в папке `events`.

<details>

# Устанавливает, будут ли события транслироваться только в том мире, в котором они происходят.
Only broadcast event message in event worlds: false
# Устанавливает минимальное время восстановления в минутах между временными событиями.
actionEventMinimumCooldownMinutes: 240
# Устанавливает, будут ли происходить события действий.
# https://github.com/MagmaGuy/EliteMobs/wiki/Creating-Custom-Events#action-events
actionEventsEnabled: true
# Устанавливает, будут ли происходить временные события.
# https://github.com/MagmaGuy/EliteMobs/wiki/Creating-Custom-Events#timed-events
timedEventsEnabled: true
```

normalizedRegionalBossBaselineHealthV3: 4.0
# Устанавливает, будут ли региональные боссы использовать нормализованную боевую систему.

normalizeRegionalBosses: true

fullHealMessage: '&2ПОЛНОЕ ИСЦЕЛЕНИЕ!'

# Устанавливает множитель, применяемый к снижению урона, когда игрок держит щит для атак ближнего боя (способности исключены).
blockingDamageReduction: 0.8

## Database.yml

`Database.yml` включает настройки для того, как вы хотите, чтобы EM обрабатывал базы данных.

<details>

<summary><b>Развернуть таблицу</b></summary>

```yml
# Устанавливает, будет ли использоваться MySQL. По умолчанию EliteMobs использует SQLite. Если вы хотите использовать MySQL, вам потребуется действительная конфигурация MySQL.
useMySQL: false
# Хост вашей базы данных MySQL.
mysqlHost: localhost
# Порт вашей базы данных MySQL.
mysqlPort: '3306'
# Название базы данных.
mysqlDatabaseName: elitemobs
# Имя пользователя для MySQl.
mysqlUsername: your_username_mysql_here
# Пароль для вашей базы данных MysSQL.
mysqlPassword: your_mysql_password_here
# Использовать ли SSL.
useSSL: true
```

</details>

## Sounds.yml

`Sounds.yml` Включает в себя настройки, которые определяют, какие звуки должны воспроизводиться при выполнении определенных взаимодействий EliteMobs. Звуковые файлы хранятся в пакете ресурсов.

<details>

<summary><b>Развернуть таблицу</b></summary>

```yml
# Устанавливает звук, который будет воспроизводиться при открытии сундука с сокровищами.
treasureChestOpenSound: elitemobs:treasure_chest.open
# Устанавливает звук, который будет воспроизводиться, когда игрок повышает ранг в гильдии.
guildRankUpSound: elitemobs:guild.rankup
# Устанавливает звук, который будет воспроизводиться, когда игрок получает престиж в гильдии.
guildPrestigeSound: elitemobs:guild.prestige
# Устанавливает звук, который будет воспроизводиться, когда игрок продвигается в квесте.
questProgressionSound: elitemobs:quest.progression
# Устанавливает звук, который будет воспроизводиться, когда игрок завершает квест.
questCompleteSound: elitemobs:quest.completion
# Устанавливает звук, который будет воспроизводиться, когда игрок отказывается от квеста.
questAbandonSound: elitemobs:quest.abandon
# Устанавливает звук, который будет воспроизводиться, когда игрок принимает квест.
questAcceptSound: elitemobs:quest.accept
```

</details>

## Wormholes.yml

`Wormholes.yml` Включает в себя настройки для телепортов червоточины EliteMobs.

<details>

<summary><b>Развернуть таблицу</b></summary>

```yml
# Устанавливает сообщение, которое появляется, когда телепорт используется для подземелья, которое не установлено.
dungeonNotInstalledMessage: '&8[EliteMobs] &cПодземелье $dungeonID &cне установлено!
  Этот телепорт не будет работать.'
# Устанавливает сообщение, которое появляется, когда червоточина используется для подземелья, которое не установлено.
defaultPortalMissingMessage: '&8[EliteMobs] &cПохоже, этот портал никуда не ведет!'
# Устанавливает, используется ли режим уменьшенного количества частиц для червоточин. Это особенно рекомендуется, если вы разрешаете клиентам Bedrock.
reducedParticlesMode: true
# Устанавливает, используют ли червоточины частицы вообще. Не рекомендуется, но может быть необходимо для действительно плохих клиентов Bedrock.
noParticlesMode: false
# Устанавливает сообщение, которое отправляется, когда игрок пытается использовать червоточину, но у него недостаточно валюты для ее использования.
insufficientCurrencyForWormholeMessage: '&8[EliteMobs] &cНедостаточно валюты! Вы
  нуждаетесь в $amount, чтобы использовать эту червоточину!'
```

</details>
