これはEliteMobsにおける体力とダメージのスケールに関する包括的な解説です。ダンジョンを設計する際の参考としてください。

カスタムボスに焦点を当てていますが、通常のEliteも同様にスケールします。

# 始める前に！

ちょっとした調整をしたいだけなら、すべてを読む必要はありません。「ダンジョンでのこのデータの使い方」セクションに進み、非常に簡略化された簡単な計算と、あらかじめ作成された推奨設定のシリーズをご覧ください！

# スケーリング（一般）

EliteMobsでは、デフォルトでボスはレベルごとに最大体力+7、ダメージ+1されます。

プレイヤーの武器はレベルごとにダメージ+7、防具はレベルごとにダメージを0.25減少させます（ただし、合計に寄与する防具スロットは4つあり、合計1となります）。最後に、プレイヤーはプレステージランクをアンロックすると最大体力を得ます。

# カスタムボスによるダメージ

カスタムボスのダメージ計算式は次のとおりです。

```
( ( ( 基本ダメージ / 2 ) * カスタムボスダメージ倍率) + ボスのレベル - プレイヤーの防具によるダメージ軽減 ) * グローバルダメージ倍率
```

## 基本ダメージ

これは次の2つのうちのいずれかになります。

1.  ボスのエンティティタイプのmobpropertiesの値。
2.  ダンジョンやその他のコンテンツを設計する際にこれを念頭に置いておくのが難しい場合があるため、カスタムボスにはnormalizedCombatフィールドがあり、EliteMobsの進行に最適な標準の5ダメージが適用されます。後のアップデートでは、normalized combatはすべての地域ボスでデフォルトで有効になります。

## カスタムボスダメージ倍率

これはカスタムボスファイルにあるダメージ倍率です。デフォルトは1です。

## ボスのレベル

ご想像のとおり、これはボスのレベルです。つまり、ボスはレベルごとにダメージが1増加します。

## プレイヤーの防具によるダメージ軽減

プレイヤーの防具はレベルごとにダメージを0.25減少させ、合計に寄与できる防具スロットは4つあります。さらに、投射物保護の各レベルは、投射物ダメージを0.025減少させます。

## グローバルダメージ倍率

これはMobCombatSettingsの倍率です。デフォルトは1です。

# カスタムボスの最大体力

カスタムボスの体力計算式は次のとおりです。

```
( 基本最大体力 + ボスのレベル * 7 ) * カスタムボス体力倍率
```

## 基本最大体力

これは次の2つのうちのいずれかになります。

1.  バニラMinecraftエンティティのデフォルトの最大体力。
2.  ダンジョンやその他のコンテンツを設計する際にこれを念頭に置いておくのが難しい場合があるため、カスタムボスにはnormalizedCombatフィールドがあり、EliteMobsの進行に最適な標準の7体力を使用します。Normalized combatはすべての地域ボスでデフォルトで有効になっています。

## レベル

これはボスのレベルです

## カスタムボス体力倍率

これはカスタムボスファイルにある倍率です。デフォルトは1です。

# カスタムボスに与えるダメージ

プレイヤーがカスタムボスに与えるダメージの計算式には、クリティカルストライクやクールダウンなど、ここで詳細に説明するには変数が多すぎます。簡略化されたバージョンは次のようになります。

```
ダメージ = 武器のレベル
```

これは、理想的な平均設定でのダメージのおおよその目安です。

# ダンジョンでのこのデータの使い方

**注意：これは`normalizedCombat`の値を使用することを前提としています！** これらの値は、ダンジョンのバランスをより簡単にするために特別に作成されました。

ボスの難しさの一部は、エンカウンターに適した体力とダメージを持つようにすることです。ただし、これはその一部にすぎず、パワーも同様に重要であることに注意してください。

以下は、カスタムボスファイルの正しい形式で示されています。

## 簡単な計算

1.  EliteMobsは、ボスと同じレベルの武器を持つプレイヤーがボスを倒すのに必要なヒット数を7としています。
    1.  つまり、2.0倍率は、プレイヤーがボスを2 x 7 = 14回ヒットする必要があることを意味します。0.5は7 / 2 = 3.5を意味します。
2.  プレイヤーの防具がボスと同じレベルであると仮定した場合、ベースラインのnormalizedCombatダメージは、1回のヒットあたり2.5ダメージまたは1.25ハートのダメージです。
    1.  つまり、2.0倍率は、プレイヤーが1回のヒットあたり3ハートを失うことを意味し、0.5倍率は、プレイヤーが1回のヒットあたり0.75ハートを失うことになります。

**プレイヤーは武器を優先する傾向があり、多くの場合、武器よりもはるかに悪い防具を使用していることに注意してください。射撃戦闘を備えた非常に特定のタイプのアリーナを設計しない限り、ヒットごとに多くのダメージを与えたくないでしょう。** コンテンツをプレイテストしてください！

## 平均的なダンジョンモブの作成

normalizedCombatのスケーリングを使用すると、非常に最小限の構成を適用できます。

```yaml
normalizedCombat: true
healthMultiplier: 1.0 #4ヒットで倒せる
damageMultiplier: 1.0 #1.5ハートのダメージ
```

**注：1.0がデフォルトであるため、何も定義しないこともできます。**

## 雑魚パックの作成

雑魚パックは多数ですが、それほど危険ではありません。

```yaml
normalizedCombat: true
healthMultiplier: 0.7 #3ヒットで倒せる
damageMultiplier: 0.5 #0.5ハートのダメージ
```

## 増援の作成

増援は非常に簡単に死ぬ必要がありますが、ダメージに関しては危険です（これらの値は4つ以上の近接エンティティに推奨されますが、実際の使用は異なる場合があります）。

```yaml
normalizedCombat: true
healthMultiplier: 0.25 #1ヒットで倒せる
damageMultiplier: 0.6 #1ハートのダメージ
```

## ミニボスの作成

ミニボスは持ちこたえ、プレイヤーにメカニックチャレンジを課す必要があります。しばらく続くスキルのテストである必要がありますが、非常に危険なものであってはなりません。

```yaml
normalizedCombat: true
healthMultiplier: 3.0 #10ヒットで倒せる
damageMultiplier: 1.2 #2ハートのダメージ
```

## ボスの作成

ボスは真の挑戦であり、ダンジョンでの積み上げの結論であり、すべてを賭けた真のスキルのテストです。エンカウンターは長く、死は差し迫った脅威である必要があります。

```yaml
normalizedCombat: true
healthMultiplier: 7.0 #23ヒットで倒せる
damageMultiplier: 1.4 #2.5ハートのダメージ
```

## ワールドボスの作成

ワールドボスは危険ですが、何よりも体力が高く、多くの増援を召喚したり、多くの範囲攻撃を持っています。これらは、エリートエンカウンターが続くべき最長の期間に、大人数のプレイヤーグループによって戦われることを目的としています。それらはそれほど致命的ではありませんが、準備不足のプレイヤーを圧倒する可能性のある多くのメカニックを備えています。

```yaml
normalizedCombat: true
healthMultiplier: 40.0 #160ヒットで倒せる
damageMultiplier: 1.5 #2.5-3ハートのダメージ
```

これらはあくまでおおまかな推奨値であり、最終的な値はエンカウンター自体に合わせて調整する必要があります。
