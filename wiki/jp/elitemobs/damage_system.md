はい、翻訳します。

これは、EliteMobs における体力とダメージのスケールに関する包括的な解説です。ダンジョンを設計する際の参考として使用してください。

カスタムボスに焦点を当てていますが、通常の精鋭も同様にスケールします。

# 開始する前に！

簡単な調整だけを行いたい場合は、これをすべて読む必要はありません。「ダンジョンでのこのデータの使用」セクションに進み、非常に簡略化された簡単な計算と、事前に作成された推奨設定のシリーズを確認してください！

# スケーリング (一般)

EliteMobs では、デフォルトでボスはレベルごとに最大体力+7、ダメージ+1を得ます。

プレイヤーの武器はレベルごとにダメージ+7、防具はレベルごとにダメージを-0.25減らします（ただし、合計に貢献する4つの防具スロットがあり、合計で1になります）。最後に、プレイヤーはプレステージランクを解除すると最大体力を得ます。

# カスタムボスによるダメージ

カスタムボスのダメージ計算式は次のとおりです。

```
( ( ( ベースダメージ / 2 ) * カスタムボスダメージ倍率) + ボスのレベル - プレイヤーの防御力減少 ) * グローバルダメージ倍率
```

## ベースダメージ

これは次の2つのうちのいずれかになります。

1. ボスのエンティティタイプの mobproperties の値。
2. ダンジョンやその他のコンテンツを設計する際にこれを念頭に置いておくのは難しい場合があるため、カスタムボスは
   normalizedCombat フィールドを持つことができ、EliteMobs の進行に最適な標準的な 5 ダメージを適用します。後のアップデートでは、すべてのリージョンボスに対して
   normalizedCombat がデフォルトで有効になります。

## カスタムボスダメージ倍率

これは、カスタムボスファイル内のダメージ倍率です。デフォルトは 1 です。

## ボスのレベル

ご想像のとおり、これはボスのレベルです。これは、ボスがレベルごとに 1 ダメージ多く与えることを意味します。

## プレイヤーの防御力減少

プレイヤーの防御力はレベルごとに 0.25 ダメージを減らし、合計に貢献できる 4 つの防具スロットがあります。さらに、投射物保護の各レベルは、投射物ダメージを
0.025 減少させます。

## グローバルダメージ倍率

これは MobCombatSettings の倍率です。デフォルトは 1 です。

# カスタムボスの最大体力

カスタムボスの体力計算式は次のとおりです。

```
( ベース最大体力 + ボスのレベル * 7 ) * カスタムボス体力倍率
```

## ベース最大体力

これは次の2つのうちのいずれかになります。

1. バニラ Minecraft エンティティのデフォルトの最大体力。
2. ダンジョンやその他のコンテンツを設計する際にこれを念頭に置いておくのは難しい場合があるため、カスタムボスは
   normalizedCombat フィールドを持つことができ、EliteMobs の進行に最適な標準的な 7 体力を適用します。 normalizedCombat
   は、すべてのリージョンボスに対してデフォルトで有効になっています。

## レベル

これはボスのレベルです

## カスタムボス体力倍率

これは、カスタムボスファイル内の倍率です。デフォルトは 1 です。

# カスタムボスに与えるダメージ

プレイヤーがカスタムボスに与えるダメージの計算式には、クリティカルストライクやクールダウンなど、ここで詳しく説明するには小さすぎる変数が多すぎます。単純化したバージョンは次のようになります。

```
ダメージ = 武器のレベル
```

これは、理想的な平均設定でダメージがどのように見えるかの大まかな目安です。

# ダンジョンでのこのデータの使用

**注意: これは `normalizedCombat` 値を使用していることを前提としています!** これらの値は、ダンジョンのバランスをより簡単にするために特別に作成されました。

ボスの難しさの一部は、エンカウンターに適切な体力とダメージを持たせることです。ただし、これはその一部にすぎず、パワーも同じくらい重要であることを覚えておいてください。

以下は、カスタムボスファイルの正しい形式で表示されます。

## 簡単なクイック計算

1. EliteMobs では、ボスと同じレベルの武器を持つプレイヤーがボスを倒すのにかかるヒット数を 7 回としています (
   修飾子がない場合)。
    1. これは、2.0 の乗数が、プレイヤーがボスを 2 x 7 = 14 回攻撃する必要があることを意味します。0.5 は 7 / 2 = 3.5 を意味します。
2. プレイヤーの防具がボスと同じレベルであると仮定した場合、ベースラインの normalizedCombat ダメージは 2.5 ダメージ、つまり
   1 回のヒットあたり 1.25 ハートのダメージです。
    1. これは、2.0 の乗数が、プレイヤーがヒットごとに 3 ハートを失うことを意味し、0.5 の乗数が、プレイヤーがヒットごとに
       0.75 ハートを失うことを意味します。

**プレイヤーは武器を優先する傾向があり、武器よりもはるかに劣る防具を頻繁に使用することを覚えておいてください。範囲攻撃のある非常に特定のタイプのアリーナを設計しない限り、ヒットごとに多くのダメージを与えたくないでしょう。
** コンテンツをプレイテストしてください!

## 平均的なダンジョンモブの作成

normalizedCombat のスケーリングを使用すると、非常に最小限の構成を適用できます。

```yaml
normalizedCombat: true
healthMultiplier: 1.0 #倒すのに4回攻撃
damageMultiplier: 1.0 #1.5ハートダメージ
```

**注: 1.0 がデフォルトであるため、これは何も定義しないこともできることを意味します**。

## ゴミパックの作成

ゴミパックは多数ですが、それほど危険ではありません。

```yaml
normalizedCombat: true
healthMultiplier: 0.7 #倒すのに3回攻撃
damageMultiplier: 0.5 #0.5ハートダメージ
```

## 増援の作成

増援は非常に簡単に死ぬはずですが、ダメージの観点からは危険を及ぼします（これらの値は 4
つ以上の近接エンティティに推奨されますが、実際の使用状況は異なる場合があります）。

```yaml
normalizedCombat: true
healthMultiplier: 0.25 #倒すのに1回攻撃
damageMultiplier: 0.6 #1ハートダメージ
```

## ミニボスの作成

ミニボスは、その場を維持し、プレイヤーにメカニックチャレンジを課す必要があります。それはしばらく続くスキルのテストであるはずですが、非常に致命的なものではありません。

```yaml
normalizedCombat: true
healthMultiplier: 3.0 #倒すのに10回攻撃
damageMultiplier: 1.2 #2ハートダメージ
```

## ボスの作成

ボスは真の挑戦であり、ダンジョンでの積み重ねの結論であり、すべてを賭けたスキルの真の試練です。エンカウンターは長く、死は常に脅威となるはずです。

```yaml
normalizedCombat: true
healthMultiplier: 7.0 #倒すのに23回攻撃
damageMultiplier: 1.4 #2.5ハートダメージ
```

## ワールドボスの作成

ワールドボスは危険ですが、何よりも体力が多く、多くの増援を召喚したり、範囲攻撃を多く行ったりします。これらは、エリートエンカウンターが持続する最も長い時間、多数のプレイヤーグループによって戦われることを目的としています。それらはそれほど致命的ではありませんが、準備不足のプレイヤーを圧倒する可能性のある多くのメカニズムを備えています。

```yaml
normalizedCombat: true
healthMultiplier: 40.0 #倒すのに160回攻撃
damageMultiplier: 1.5 #2.5-3ハートダメージ
```

これらは大まかな推奨値であり、最終的な値はエンカウンター自体を考慮して調整する必要があります。
