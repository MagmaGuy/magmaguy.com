EliteMobsにおける体力/ダメージのスケーリングがどのように機能するかの包括的な内訳です。ダンジョンを設計する際の参考資料として使用してください。

カスタムボスに焦点を当てていますが、通常のエリートも同じようにスケーリングします。

# 始める前に！

簡単な調整だけをしたい場合は、これをすべて読む必要はありません。「このデータをダンジョンに利用する」セクションにスキップして、非常に簡略化された簡単な計算と、事前に用意された推奨設定のシリーズを確認してください！

# スケーリング（全般）

EliteMobsでは、デフォルトで、ボスはレベルごとに最大体力が7増加し、ダメージが1増加します。

プレイヤーの武器はレベルごとにダメージが7増加し、防具はレベルごとにダメージを0.25軽減します（ただし、合計に寄与する防具スロットは4つあり、合計で1になります）。最後に、プレイヤーはプレステージランクを解除するにつれて最大体力を獲得します。

# カスタムボスが与えるダメージ

カスタムボスのダメージ計算式は以下の通りです。

```
( ( ( ベースラインダメージ / 2 ) * カスタムボスダメージ乗数) + ボスのレベル - プレイヤーの防具による軽減 ) * グローバルダメージ乗数
```

## ベースラインダメージ

これは以下の2つのうちのいずれかです。

1.  ボスのエンティティタイプのmobpropertiesにある値。
2.  ダンジョンや他のコンテンツを設計する際にこれを覚えておくのは難しい場合があるため、カスタムボスはnormalizedCombatフィールドを持つことができます。これは標準的な5ダメージを適用し、EliteMobsの進行に最適なスケーリングを提供します。後続のアップデートでは、すべてのリージョンボスに対してnormalized combatがデフォルトで有効になります。

## カスタムボスダメージ乗数

これはカスタムボスファイルにあるダメージ乗数です。デフォルトは1です。

## ボスのレベル

ご想像の通り、これは単にボスのレベルです。これは、ボスがレベルごとにダメージを1多く与えることを意味します。

## プレイヤーの防具による軽減

プレイヤーの防具はレベルごとにダメージを0.25軽減し、合計に寄与できる防具スロットは4つあります。さらに、飛び道具耐性のレベルごとに、飛び道具ダメージを0.025軽減します。

## グローバルダメージ乗数

これはMobCombatSettingsにある乗数です。デフォルトは1です。

# カスタムボスの最大体力

カスタムボスの体力計算式は以下の通りです。

```
( ベースライン最大体力 + ボスのレベル * 7 ) * カスタムボス体力乗数
```

## ベースライン最大体力

これは以下の2つのうちのいずれかです。

1.  バニラMinecraftエンティティのデフォルトの最大体力。
2.  ダンジョンや他のコンテンツを設計する際にこれを覚えておくのは難しい場合があるため、カスタムボスはnormalizedCombatフィールドを持つことができます。これは標準的な7体力を適用し、EliteMobsの進行に最適なスケーリングを提供します。normalized combatはすべてのリージョンボスに対してデフォルトで有効になっています。

## レベル

これはボスのレベルです。

## カスタムボス体力乗数

これはカスタムボスファイルにある乗数です。デフォルトは1です。

# カスタムボスに与えるダメージ

プレイヤーがカスタムボスに与えるダメージ計算式には、クリティカルヒットやクールダウンなど、ここで詳細に説明するにはあまりにも多くの小さな変数があります。簡略化されたバージョンは以下のようになります。

```
ダメージ = 武器のレベル
```

これは、理想的な平均的な設定でダメージがおおよそどのように見えるかを示しています。

# このデータをダンジョンに利用する

**注：これは`normalizedCombat`の値を使用していることを前提としています！** これらの値は、ダンジョンのバランス調整をより簡単にするために特別に作られました。

ボスの難易度の一部は、エンカウンターに対して適切な体力とダメージを持たせることです。ただし、これはその一部にすぎず、パワーも同様に重要であることを覚えておいてください。

以下は、カスタムボスファイルの正しい形式で提示されています。

## 簡単な早見計算

1.  EliteMobsでは、プレイヤーがボスと同じレベルの武器でボスを倒すのに必要なヒット数は7ヒットとされています（修飾子がないと仮定）。
    1.  これは、乗数が2.0の場合、プレイヤーはボスに2 x 7 = 14回ヒットする必要があることを意味します。0.5の場合は7 / 2 = 3.5回です。
2.  プレイヤーの防具がボスと同じレベルであると仮定した場合の、ベースラインnormalizedCombatダメージは、1ヒットあたり2.5ダメージ、または1.25ハートのダメージです。
    1.  これは、乗数が2.0の場合、プレイヤーは1ヒットあたり3ハートを失い、乗数が0.5の場合、プレイヤーは1ヒットあたり0.75ハートを失うことを意味します。

**プレイヤーは武器を優先する傾向があり、防具は武器よりもはるかに劣ることが多いことに注意してください。遠距離戦闘を伴う非常に特定のタイプのアリーナを設計しない限り、1ヒットあたり多くのダメージを与えたくないでしょう。** コンテンツをプレイテストしてください！

## 平均的なダンジョンモブの作成

normalizedCombatのスケーリングを使用すると、非常に最小限の設定を適用できます。

```yaml
normalizedCombat: true
healthMultiplier: 1.0 # 倒すのに4ヒット
damageMultiplier: 1.0 # 1.5ハートのダメージ
```

**注：1.0はデフォルトであるため、何も定義しないことも可能です**。

## 雑魚敵パックの作成

雑魚敵パックは数が多いですが、それほど危険ではありません。

```yaml
normalizedCombat: true
healthMultiplier: 0.7 # 倒すのに3ヒット
damageMultiplier: 0.5 # 0.5ハートのダメージ
```

## 増援の作成

増援は非常に簡単に死ぬべきですが、ダメージの面で危険をもたらします（これらの値は4体以上の近接エンティティに推奨されます。実際の使用は異なる場合があります）。

```yaml
normalizedCombat: true
healthMultiplier: 0.25 # 倒すのに1ヒット
damageMultiplier: 0.6 # 1ハートのダメージ
```

## ミニボスの作成

ミニボスは持ちこたえ、プレイヤーにメカニクス的な挑戦をもたらすべきです。それは少しの間続くスキルテストであるべきですが、極端に致命的なものではありません。

```yaml
normalizedCombat: true
healthMultiplier: 3.0 # 倒すのに10ヒット
damageMultiplier: 1.2 # 2ハートのダメージ
```

## ボスの作成

ボスは本当の挑戦であり、ダンジョンでの盛り上がりの締めくくりであり、全てがかかった本当のスキルテストです。エンカウンターは長く、死は常に存在する脅威であるべきです。

```yaml
normalizedCombat: true
healthMultiplier: 7.0 # 倒すのに23ヒット
damageMultiplier: 1.4 # 2.5ハートのダメージ
```

## ワールドボスの作成

ワールドボスは危険ですが、何よりも多くの体力を持っており、多くの増援を召喚したり、多くの範囲攻撃を持っています。これらは、エリートエンカウンターが続くべき最も長い時間、大規模なプレイヤーグループによって戦われることを意図しています。それほど致命的ではありませんが、準備不足のプレイヤーを圧倒する可能性のある多くのメカニクスを持っています。

```yaml
normalizedCombat: true
healthMultiplier: 40.0 # 倒すのに160ヒット
damageMultiplier: 1.5 # 2.5-3ハートのダメージ
```

これらはあくまで大まかな推奨値であり、最終的な値はエンカウンター自体に合わせて調整されるべきです。
